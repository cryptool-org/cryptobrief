1 Advanced Search Patterns

The following table shows some valid advanced search patterns and a description of what these patterns match.

Advanced Search Pattern Description
%1 the entire text
a%1b an ’a’, followed by a string which does not include a ’b’, followed by a ’b’
%1a%2 a string which does not include an ’a’, followed by an ’a’, followed by any string
a%1%%1 an ’a’, followed by a string which does not include ’%1’, followed by ’%1’

By putting a % in front of a variable, the variable is escaped. So, %%1 does not represent a variable but the string %1. Patterns starting or ending with a variable have to be used carefully, as those patterns possibly could match the entire text.

There are three types of invalid search patterns. A pattern is invalid if there is no variable used in the pattern, two consecutive variables without delimiter in between are used, or the same variable is used more than once. Some examples of invalid search patterns are:

Advanced Search Pattern Description
abc Missing variable in the search pattern
%1%2 Two variables without delimiter in between
%1b%1 Variable %1 used more than once

If no variable is used in the search pattern, a standard search must be used instead of an advanced search. Also, two consecutive variables are invalid because it is not clear which variable should match which content. This also applies to the last invalid search pattern from above table, as reusing the same variable makes it unclear what this variable should match (the first match, the second match, or a combination of both). Advanced search patterns can be also converted into a corresponding regular expression.

2 Advanced Replacements Patterns

For replacement patterns there are less restrictions than for search patterns. If the search pattern a%1c is applied to the text ’abc’, the replacement patterns shown in below table are valid examples. The resulting replacement strings are also shown in this table.

Advanced Replacement Pattern Resulting Replacement String
abc ’abc’
%1 ’b’
c%1a ’cba’
%1%1%1 ’bbb’
%%0%1%%1 ’%0b%1’

So, replacement patterns do not need to comprise variables at all, and it’s allowed to use the same variable more than once in the replacement pattern. Invalid replacement patterns are patterns that comprise variables which were not used in the search pattern, for example a%2b or %1%9 in above context. So, all replacement patterns which do not comprise unused variables are valid. One important note here is that not only the variable but the entire match of the search pattern is replaced with the according replacement string. In the example above, the search pattern matches the entire text, therefore the entire text is replaced with the according replacement string. In a more realistic scenario, the search starts from a given position in the text (the position of the cursor), and typically matches only a substring of the text.

3 Balancing

The advanced search approach also supports a configurable balancing mechanism which properly match balanced constructs such as nested if-statements in a Java code fragment. The balancing configuration allows so-called matching pairs to be specified by the user. A valid matching pair configuration starts with a string which does not comprise dot (.) and comma (,) as these two symbols are used as delimiters in the configuration. This string acts as an opening tag and must be followed by ... and another string without dot and comma. The second string acts as a closing tag. These two strings form a so-called matching pair. So, if an opening tag is found in the text, the according closing tag must be found subsequently in order to have a proper balanced string. To configure multiple matching pairs, a comma is used to delimit the matching pairs from each other. Additionally, it should be possible to include variables in the balancing configuration, to support matching pairs which content is not fixed. These are some examples of valid matching pair configurations, here referring to the LaTeXsyntax as an example:

  • (...)
  • (...), [...], {...}
  • open...close
  • \begin{%1}...\end{%1}
  • start[%1-%2-%3]...end(%3-%2-%1)
  • (...), [...], {...}, \(...\), \[...\], \{...\}, \begin{%1}...\end{%1}

The last item represents the default balancing configuration which is used if no configuration is specified by the user. If an empty balancing configuration is specified, the balancing functionality is disabled. If variables are used in the matching pairs, the content of the variables used in the opening and closing tags have to be identical. Also, same as for the variable constraints in the search pattern, the variables in the matching pair configuration also have to be delimited by at least one character and the same variable must not be used more than once per tag. Additionally, the exact same variables have to be used in the opening and closing tag of a matching pair. The following examples are invalid matching pair configurations:

Invalid Configuration Comment
(..) Invalid delimiter between tags
(),[],{} Missing delimiter between tags
(...) [...] Invalid delimiter between matching pairs
open[a,b]...close[a,b] Tags must not include matching pair delimiter
\begin{%1}...\end{} Missing variable in closing tag
start[%1-%2-%3]...end(%6-%5-%4) Opening and closing tags use different variables
\begin[%1%2]...\end(%1%2) Missing delimiter between variables
\begin[%1-%1]...\end(%1-%1) Variable %1 used more than once per tag

4 Error Recovery

The most important error messages and possible actions to recover are shown next.

  • Invalid search pattern! Missing delimiter between variables

    Either remove one of the variables from the search pattern or insert a delimiter string between variables

  • Invalid search pattern! Variable has been used more than once

    Remove duplicate variables from the search pattern

  • Invalid search pattern! No variable used in the search pattern

    Either switch to Standard Search mode or insert at least one variable in the search pattern

  • Variable from replace text has not been used in search pattern

    Check naming of variables in replacement pattern, remove or rename variables which are not used in the search pattern

  • Bad matching pair configuration

    Check matching pair configuration constraints from Section 3

  • Matched string is unbalanced

    This error is shown because the Show balancing error option is enabled and an unbalanced string was found in the text. Either disable this option or check the document for errors.
